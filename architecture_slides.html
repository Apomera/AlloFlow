Core Principle

The Monolithic Root

AlloFlow operates on a radical architectural principle: Extreme consolidation. By housing the entire core engine in a single massive file (AlloFlowANTI.txt), we achieve zero-latency context propagation between deeply integrated UI features.

  • Zero Router Overhead: True single-page dominance without route transitioning latency.
  • State Monopoly: All critical educational states live in root React Context, immediately accessible by the AI generation loops.
  • Deterministic Automation: Codebase mutations skip conventional build-trees, relying on deterministic Python Ast-patching for massive overhauls.

UX Orchestration

The Deterministic Tour Engine

AlloFlow features an integrated AI copilot that physically interacts with the application. Instead of basic static tooltips, the Tour Engine parses AI-generated JSON instructions to physically open sidebars, switch active modes, and scroll to target DOM elements.

  • data-help-key Binding: Every interactive element is tagged. The Tour Engine queries the DOM to find coordinates and bind the Allobot avatar UI absolute to the target.
  • Progressive Revelation: The engine pauses TTS playback, waits for DOM rendering stability, and ensures the target element is visible before executing the next instruction sequence.

Bimodal Pipeline

Bridge Mode & Synchronous TTS

Deep Translation

Unlike Google Translate over-the-top styling, AlloFlow translates pedagogical text at the AI generation root. The math problem is generated natively in the target language.

The Orchestration Gap

To prevent reading delays, TTS audio (via experimental Preview endpoints or Web Speech) is fetched concurrently while the translated text streams to the UI context.

Proxy Audio Caching

Base64 TTS audio strings are captured and stored in highly-optimized Proxy wrappers to prevent memory leak invariant violations on re-renders.

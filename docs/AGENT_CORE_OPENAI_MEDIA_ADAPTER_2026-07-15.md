# Agent Core OpenAI media adapter foundation

Date: 2026-07-15

## Status

Implemented and tested, but disabled and unregistered. The adapter is not
loaded by AlloFlow, Gemini Canvas, the desktop runtime, or the MCP server.

`OPENAI_MEDIA_EXECUTION_DEFAULT_ENABLED` is `false`.

## Official API contract used

The implementation follows OpenAI's current GPT Image 2 documentation:

- model: `gpt-image-2` (or an explicitly pinned GPT Image 2 snapshot);
- generation: `POST https://api.openai.com/v1/images/generations` with JSON;
- editing: `POST https://api.openai.com/v1/images/edits` with multipart
  `image[]` fields for source/reference images;
- GPT image output is read from `data[0].b64_json`;
- supported output sizes are normalized to `1024x1024`, `1536x1024`, or
  `1024x1536`;
- quality is normalized to `low`, `medium`, or `high`;
- output format is `png`, `jpeg`, or `webp`;
- `input_fidelity` is intentionally omitted for GPT Image 2 because the model
  always processes image inputs at high fidelity and does not permit changing
  that option.

References:

- https://developers.openai.com/api/docs/guides/image-generation
- https://developers.openai.com/api/reference/resources/images/methods/generate
- https://developers.openai.com/api/reference/resources/images/methods/edit
- https://developers.openai.com/api/docs/models/gpt-image-2

## Credential boundary

The adapter never accepts an API key. A trusted runtime must inject
`fetchAuthorized(url, request, context)`. That credential broker adds
authorization after the agent-visible request has been validated. API keys
therefore never enter a MediaRequest, Blueprint, MCP response, artifact,
provenance record, or audit payload.

## Required preflight

Before a provider request, the guarded runtime requires:

- valid MediaRequest 1.0;
- explicit `openai-api` provider/model permission;
- `allowMeteredUsage: true`;
- an injected conservative cost upper bound;
- an approved maximum cost and operation count;
- an injected runtime authorization decision;
- an injected actual-cost calculator;
- an available alt-text function when accessibility requires alt text.

A failed preflight performs no fetch.

## Asset behavior

For editing, source and reference bytes are obtained from the strict managed
asset store through runtime-authorized adapter reads. The agent sees only
opaque handles. Generated bytes are imported back into the store, and the
result contains only managed metadata, alt text, provider/model labels, and
accounted usage.

## Remaining production gates

1. Decide and test the alt-text provider policy. If alt text uses a metered
   vision call, its upper-bound and actual costs must be part of the parent job
   budget; an included/local alt-text implementation can avoid that extra
   metered operation.
2. Add decoded-output byte limits before processing very large responses.
3. Add retry policy that never exceeds the approved operation count.
4. Add pinned price-table data with an effective date, plus a refusal when the
   table is stale.
5. Test against an approved non-production OpenAI project. Current tests use a
   fake credential broker and make no network calls.
6. Complete educational quality evaluation against Gemini before choosing any
   default provider.
7. Keep MCP media tools disabled until permission and audit review is complete.

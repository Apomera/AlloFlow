# AlloFlow — Shippable Handoff (2026-07-15)

## ⚠️ CRITICAL fatal-crash fix (2026-07-15, later)
`AlloFlowContent` crashed on mount with `ReferenceError: Cannot access 'useLowQualityVisuals'
before initialization` (whole app down via ErrorBoundary). Cause: `useLowQualityVisuals`'s
`useState` was declared ~10k lines BELOW two render-time `useEffect` dependency arrays that read
it (adventure autosave ~L10629, project sync ~L13677) — a temporal-dead-zone error. **Fixed** by
moving the declaration up into the top-level `useState` cluster (~L6600), before first use, in
`AlloFlowANTI.txt` (canonical; `build.js` regenerates `App.jsx` at deploy). Pre-existing/latent
(present in the live/`origin` code) — NOT introduced this session. **The deploy gates did NOT
catch this** (`check_free_vars` only flags *undeclared* vars, not use-before-declaration of a
declared `const`). A **second** instance surfaced right after deploy: `stopPlayback` (a `const`
function read at render by the `_personasLiveRef.current = { …, stopPlayback, … }` live-ref capture
~1.2k lines before its definition) — fixed by making it a **hoisted `function stopPlayback(){}`**.
**FIXED the gate gap:** added `dev-tools/check_tdz_render.cjs` (scope-aware Babel-AST gate — fails if
any `const`/`let` in `AlloFlowContent` is read at render before its declaration) and wired it into
`deploy.sh` Step 0.6. It found exactly these two and is clean after the fixes. Canvas users get the
fix by re-pasting the updated `AlloFlowANTI.txt`; CDN/prismflow via deploy. (Likely why the pilot's
main app seemed fine: the prismflow build transpiles `const`→`var` (hoisted, no TDZ), while the
**Gemini Canvas** surface preserves `const` and crashed.)
